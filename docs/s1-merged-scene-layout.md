# 第一季合并大场景 · 布局设计（s1-hood P1）

> 状态：P1 设计定稿（2026-07-21，老板拍板故事组合 + 布局朝向）。
> 上游：`season-1-outline.md §2.1`（合并大场景策略）、`gridtiles-presets-runtime`（100 格地基）。
> 下游：P2 按本表画 100 格地形 + 注册场景/portal；P3-P4 往「穿林小径 + 外婆家」分区填《小红帽》。

## 0. 拍板结论（老板 2026-07-21）

- **进合并场景的 4 个故事**：三只小猪（村庄核心，已上线）+ 小红帽（穿林小径 + 外婆家）+
  白雪公主（森林深处·七矮人，**先预留分区**）+ 龟兔赛跑（边缘跑道，**先预留分区**）。
  绿野仙踪（奥兹黄砖路，主题「远方」）**另置场景**，不进本合并图。
- **布局朝向**：**村庄靠近端、森林向远处深入**。近端（家）= 村庄核心，越往远（深）越是没去过的地方
  （穿林小径 → 外婆家 → 森林深处七矮人）。这是「走进没去过的地方」惊喜弧的地形化（大纲 §6）。
- **本计划（s1-hood）只填《小红帽》一册**：村庄核心复用现有 village 地貌语汇，白雪/龟兔两区
  **只画地形骨架、留空**，之后各自成册再填内容。

## 1. 坐标系约定（100×100，TILE_SIZE=2，world span 200 单位）

- `x ∈ [0,99]`：西 → 东。
- `z ∈ [0,99]`：**z 小 = 森林深处（远/深），z 大 = 村庄近端（家）**。
  （即俯视图上「上=深、下=家」，与老板选定的布局预览一致。）
- 环面世界首尾循环（`WorldGrid`），但本场景四条边都是森林/山体，不设计穿接缝的动线。

## 3. 关键锚点（P2 画地形 / P3 放 POI 用）

- **村北林口**（小径起点，村庄↔森林的动线咽喉）：tile ≈ `(60, 58)`。
- **外婆家门**（`guide_to` 引路终点 POI）：tile ≈ `(71, 31)`，POI trigger 名建议 `poi_grandma`。
- **玩家出生点**：村庄广场内，tile ≈ `(50, 85)`（近端，背对森林，面朝村子）。
- **七矮人小屋位**（预留 POI）：tile ≈ `(32, 14)`。
- **跑道起点/终点**（预留 POI）：`(90, 92)` / `(90, 14)`。

## 4. 复用 / 新建边界（老板 §2.1 铁律）

- **复用**：引擎（gridtiles 100 格运行时化、占用图、chunk 流送、跨场景寻路/portal 休眠引擎）、
  管线（`export_terrain.gd` → `scene_compose.gd` → POST /admin/scenes、PackRegistry 主题包、SDF 树/篱笆物件、贴纸）。
- **新建（内容全新，不复用退役 forest 地形）**：100 格地貌 paint、各分区塑形、外婆家篱笆/小屋布景、
  小径树列。**村庄核心的地貌语汇**（广场/辐路/水井样式）可参照现有 village `_paint()`，但坐标要重排进 z∈[58,99] 近端带。

## 5. ⚠️ 待老板拍板：村庄核心怎么接现有 village（决定 P2 走法）

合并场景与「已上线的 village + 三只小猪」的关系有两条路，直接决定 P2 的规模与风险：

- **A（portal 桥接·先落地小红帽）**：合并场景是**新场景** `village_forest`，村庄核心只画个
  「林口 + 出生小广场」的入口壳，**小猪/现有 village 完全不动**；从现有 village 边缘加一对
  portal 走进合并林场景玩小红帽。最小改动、最快验《小红帽》，代价是暂时仍有一次「切场景」
  （与 §2.1「不切场景」的终态理想不完全一致，但 P2 任务本就写了「注册场景/portal」）。
- **B（全量合并·奔 §2.1 终态）**：合并场景**成为世界主场景**，把村庄 + 三只小猪的
  POI/角色/出生点**迁进来**，村庄核心画全，无 portal、无切场景。最贴 §2.1 理想，
  但要迁移已上线的小猪内容 + 重锚出生点/POI，风险与工作量大得多。

**忠犬建议过 A，老板 2026-07-21 拍板 B（全量合并为主场景）**：合并场景 `village_forest`
成为世界主场景，村庄核心画全（复用 village 地貌语汇、重排进 z∈[58,99] 近端带），
三只小猪的角色/POI/出生点迁进来，**无 portal、无切场景**——最贴 §2.1「同一个世界走到不同区
撞不同故事」的终态。

### 5.1 B 的落地拆解（P2-P4 据此）

- **P2**：产 100 格 `village_forest` 地形（村庄核心画全 + 森林各分区骨架）→ POST /admin/scenes
  注册为**世界主场景**（替换/顶替现有单场景 village 的地位）；出生点落村庄广场（≈(50,85)）；
  客户端 headless 进得去、不崩。小猪角色/POI 的具体迁移随后。
- **P3-P4**：填《小红帽》分区内容 + 迁三只小猪的 roster/POI 进合并场景（复用 createCharacter/POI 路径）。
- **风险点**：迁移已上线小猪内容 + 重锚出生点/POI，比 A 大；分步落，每步 headless 验不崩。
