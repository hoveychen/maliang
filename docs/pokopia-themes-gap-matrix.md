# Pokopia 观感扩全主题——13 主题差距取证矩阵（P1）

pokopia-themes P1 产物。对 13 个主题种子场景逐个截图（`test/test_visual_theme.gd`，
机位=修复 look_at 后的地标取景，光照与 world Pokopia 化 P1 值对齐），
对照 Pokopia 五语法（平色 / 行带 / 盖帽 / 圆角 AO / 几何装饰）打分。
截图在 wiki `art/pokopia-themes-gap-matrix`（本地 `shots_themes/`，不入库，与
`screenshots/pokopia/` 惯例一致）。

## 先说结论

1. **P1-P5/P7 的 shader 语法门控确实已全主题生效**（截图实证）：所有有机地表 mound 有
   波浪盖帽、所有台阶有圆角 AO、所有水体（礁湖/冰湖/泥潭/护城河/喷泉/景观池）有
   白泡沫描边。上一棒的「逐层门控覆盖 40 层」声称成立。
2. **真正的差距不在语法缺失,在三件事**：
   - **零装饰**（P6 散布只认 `T_GRASS`，`terrain_deco.gd:36`）——11 个非草地主题一株
     散布装饰都没有，是「死寂感」的头号来源；
   - **烘色层色板苍白**——非草地层大都 tint=WHITE、颜色烘在水彩贴图里
     （`terrain_textures.gd:146-173`），贴图均值普遍灰粉，平色收敛后整图像蒙灰纱。
     村庄的鲜绿是 tint 层，烘色层没有等价的「敢用色」；
   - **基底层漫灌**——多数种子图 60-80% 面积是同一基底层（罗马/中国=青石板、
     城市=沥青、侏罗=干裂土、海底=细沙、冰雪=雪原），Pokopia 的地面是拼布,
     我们的是地毯。这是地形数据问题（P4），shader 救不了。

## 三类修法与归属

### P2（装饰散布扩层）——收益最大
按地表层分发装饰组，复用 P6 管线（纯函数落点 + 程序化 mesh + MultiMesh 合批，
顶点预算 ≤300/mesh）。按截图饥渴程度排序：
1. seafloor：海草簇（海草层）、珊瑚枝/贝壳（珊瑚砂层）——海底没有植被最出戏;
2. jurassic：蕨叶丛（蕨类层）、枯枝/火山石子（火山岩/碎石层）;
3. icesnow：冰晶簇（冰面/压实雪）、小雪堆（雪原）
   ——落地时雪堆方案实测阵亡（白几何叠白雪隐身只剩蓝脚圈），改霜枯草簇
   （深秸秆基+霜白尖，Pokopia 雪原本来就是枯草+石头语法）;
4. medieval：麦茬束（农田垄）;
5. 室内四主题与罗马/城市/中国：装饰更该来自 items（家具/道具），terrain 散布保持克制,
   最多给 lawn_grid 一档小草芽。
密度全部低于村庄档（村庄 +11.9% 三角已是预算大头，真机未 benchmark）。

### P3（逐层调参）——修「苍白」
- 烘色层平色增饱和：flats 公式对烘色层收敛到贴图均值（灰粉），需要给
  `_compute_layer_flats` 加逐层饱和度/明度推档，或直接给苍白层改非白 tint。
  首批目标：蕨类(调绿)、海草(调青绿)、草坪格(调草绿)、地毯红(提饱和)、
  拼图垫(提饱和)、雪三层(拉开明度/加冷调差)。
- 侧壁色相统一：中世纪城台/室内平台侧壁读蓝灰，与顶面暖色失配——查侧壁层
  采样与环境冷光的叠加，必要时侧壁色相向顶面层靠。
- wall_strata 在 2-3 级 mound 上不可读：要么加幅度、要么接受（行带本来是给高崖的）。

### P4（地块种类增减提案）——修「漫灌」与「破洞」
证据已足，清单另出（P4 任务）。两个已实锤的候选：
- 室内中央台顶面换对比层（灶台顶=格纹/木、玩具台顶=拼图垫或地毯），厨房料理岛
  （木顶）立得住证明了对比顶面就是答案;
- 基底漫灌的主题补 1-2 种过渡拼布层或加大现有点缀层占比（涉服务端种子重导,
  逐项带成本给老板拍板）。
